# P20 — 핵심 모듈 테스트 커버리지 강화

## 목표

Insight 분석에서 식별된 테스트 공백을 해소한다.
**단, 외부 의존성(git 프로세스, 실제 파일시스템, tokio 런타임)이 필요한 통합 경로는 제외**하고,
순수 로직 추출 + 단위 테스트 방식으로 접근한다.

## 배경

- 이슈 1: Command 레이어 테스트 부재 — `ingest.rs`의 세션 분류 로직이 테스트 없음
- 이슈 2: Vault 하위 모듈 테스트 부재 — `index.rs` 문자열 삽입, `git.rs` diff 파싱 로직
- 이슈 4(완료): `store/db.rs` CRUD 테스트 9개 추가 (커밋 b22bf23)

## 대상 외 항목

- `#[ignore]` 테스트 9개 (kiwi/ORT/OpenAI) — 외부 모델/API 의존, CI 캐싱 인프라 없이 활성화 불가 → 현행 유지
- `sync.rs` 전체 워크플로우 — git + DB + 파일시스템 통합 의존, 단위 테스트 불가

## Subtasks

| Task | 대상 | 접근법 | 테스트 수 |
|------|------|--------|-----------|
| Task 01 | `vault/index.rs` | `build_entry_line()` + `insert_into_content()` 추출 + 테스트 | 5 |
